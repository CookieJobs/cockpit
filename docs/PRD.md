# 拾光 · 产品需求文档 (PRD)

> 版本: v0.2 · 更新日期: 2026-07-09 · 状态: 内部讨论稿
> 品牌: 拾光 (Shiguang) · 仓库: `CookieJobs/shiguang`

---

## 0. TL;DR

**拾光** 是给互联网产品/运营/技术人员的"工作流记忆外挂"。用对话式 agent 帮你记录任务、完成时沉淀成就，季度述职前不再抓瞎。

**一句话定位：** 你的个人项目驾驶舱——不替你干活，替你记住你干过什么、要干什么。

**为什么是现在：**
- LLM 长上下文 + Function Calling 成熟，"对话式 agent + 结构化数据"成为可能
- Notion AI / 飞书 AI 在做"通用 AI 笔记"，但没人专门解决"打工人多项目并行 + 述职/周报高频痛点"
- 国内还没有"专门给打工人做的工作流 agent"产品

**关键差异化（4 个护城河）：**
1. **完成即沉淀** — 任务做完不是消失了，而是沉淀成成就资产，越用越值钱
2. **本地优先** — 默认数据存本机，隐私可控；云端订阅是 opt-in
3. **中文场景深耕** — 对接飞书/钉钉/微信生态，理解国内工作流
4. **单人独立工具** — 不和飞书/Notion 抢团队协作，专注"个人项目管理"

## 1. 问题与机会

### 1.1 用户痛点（4 个真实场景）

| 场景 | 典型抱怨 | 频次 |
|---|---|---|
| **多项目并行，状态在脑子里** | "同时跟 4 个项目，每个项目做到哪一步了？" | 每天 |
| **述职前抓瞎** | "明明做了很多事，要总结时想不起做了什么" | 季度 |
| **周报靠回忆** | "周五要写周报，对着空气想半小时" | 每周 |
| **脑内待办清不掉** | "脑子里一堆事，AI 帮我倒出来整理成清单" | 每天 |

### 1.2 为什么是现在

- **技术成熟：** Claude Sonnet 4 / GPT-4o tool calling 稳定，Qwen2.5 本地部署轻量
- **认知到位：** 飞书 AI / 钉钉 AI 已经教育了"AI 助手"心智，用户不需教育
- **市场空白：** 国内打工人多项目管理+述职场景，**没有专门产品**
- **海外验证：** Granola / Reflect / Sunsama 跑通了"AI 增强工作流"付费意愿（人均 $10-30/月）

### 1.3 为什么别人没做成（也是我们的机会）

- **Notion AI** 通用笔记 + AI，不专做任务管理 → 任务颗粒度不够
- **飞书 AI** 锁在飞书生态 → 个人独立用户用不上
- **滴答清单** 任务管理强，无 LLM → 不能对话式拆解
- **Cursor** 开发专用 → 不通用
- **超级简历 / Rezi** 只做 CV → 不做"日常记录+沉淀"积累

**我们能赢的窗口期：6-12 个月。** 之后飞书/钉钉/Notion 都可能补这个场景，但他们的产品哲学是"协作"，我们是"个人"——这是定位差。

### 1.4 不做什么（砍掉的功能）

- ❌ **不做"通用 AI 笔记"**（打不过 Notion AI / 飞书 AI）
- ❌ **不做"主动操作电脑"**（Computer Use 路线，隐私雷 + 合规风险）
- ❌ **不做"主动发邮件/IM"**（合规问题 + 用户不放心）
- ❌ **不做"通用知识管理"**（Obsidian / Logseq 已经做了）
- ❌ **不做"个人 PKM / 闪念笔记"**（flomo / 锤子便签已经做了）
- ❌ **不做"团队版 / 团队协作"**（飞书/钉钉/Notion 已经做了，进去必死；个人订阅专注做透）
- ❌ **不做"番茄钟 / 时间块管理"**（专注任务管理，工具定位不同）

✅ **只做一件事：个人任务管理 + 完成时沉淀成就。** 单一场景打透。

## 2. 目标用户

### 2.1 主要用户画像：互联网产品经理"小薇"（即种子用户本人）

| 维度 | 描述 |
|---|---|
| 角色 | 3-5 年经验的产品经理，BAT/TMD 或独角兽 |
| 典型一天 | 6 个会议 + 20 条 IM + 3-4 个并行项目（功能迭代/调研/跨部门协作） |
| 核心痛点 | 状态在脑子里、述职前抓瞎、周报靠回忆 |
| 现有方案 | Notion（团队共享）+ 飞书任务（团队协作）+ 自己脑子 |
| 设备 | macOS 主力 + iPhone 备用 |
| 愿意付费 | ¥19-29/月 个人订阅 |
| 决策影响 | 单人决策，不需老板批准 |

### 2.2 次要用户画像：技术 TL"老张"

| 维度 | 描述 |
|---|---|
| 角色 | 5-8 年技术 leader，管 3-8 人小组（**注：本版本不做团队版，老张是产品未来的潜在用户，不是 MVP 目标**） |
| 典型一天 | 自己写代码 + 跟进 5 个项目 + 1v1 辅导 |
| 核心痛点 | 个人产出和团队产出混在一起，述职时区分不清 |
| 现有方案 | Jira（团队）+ 自己脑子 |
| 愿意付费 | ¥29/月（个人订阅版，不涉及团队） |

### 2.3 第三用户画像：内容运营/PMM"阿伦"

| 维度 | 描述 |
|---|---|
| 角色 | 2-4 年内容运营/PMM，独立产出为主 |
| 典型一天 | 写稿 + 选题 + 跨部门对接 + 数据复盘 |
| 核心痛点 | 周报/月报"做了什么"难总结，简历更新没素材 |
| 现有方案 | Notion / 石墨文档 + 自己脑子 |
| 愿意付费 | ¥9-19/月（频次低于产品/技术） |

### 2.4 不做的用户

- ❌ **蓝领/体力劳动者** — 场景不匹配
- ❌ **学生** — 频次太低、付费意愿弱
- ❌ **高管/VP** — 有秘书/团队，不需要个人工具
- ❌ **传统行业文员** — 场景是流程化任务，不是项目并行
- ❌ **团队/企业用户**（本版本明确不做）

### 2.5 Dogfooding 策略（重要的产品方法论）

**种子用户 = 创始人本人**。本产品采用 dogfooding 优先策略：

- **0-12 周**：创始人每天使用，把真实使用中遇到的痛点作为最高优先级需求
- **3 个月后**：开放给 10-20 个目标用户试用，收集反馈
- **6 个月后**：根据真实使用数据决定是否扩大用户群

> **核心原则**：产品是"自用品的副产品"，不是"做出来卖别人"。先把自用体验做到极致，再考虑对外。

## 3. 核心场景（MVP 5 个）

### 场景 1：倒事 → 拆解 → 落地

**触发：** 用户随口说"我今天要做这些事：写周报、修登录 bug、准备演讲 PPT（等设计出图）"

**完整流程：**
1. LLM 识别 3 件事
2. 智能判断项目维度（"写周报"是日常、"修 bug"是"Q3 产品迭代"项目、"PPT"是"季度复盘"项目）
3. 自动创建 2 个新项目 + 3 个任务
4. 给每件事建议：优先级（高/中/低）、截止日期、下一步动作、阻塞标记
5. 用户对话调整： `"修 bug 优先级最高"` → `update-task` 改 priority
6. 用户对话调整： `"PPT 等设计稿，先 block"` → `update-task` 改 blocked=true
7. 用户确认： `"好了"` → `confirm-drafts` 清除草稿高亮

**成功标准：** 60 秒内把 5 件事拆解完成，**用户只确认不手动**。

### 场景 2：完成即沉淀

**触发：** "修 bug 那个事做完了"

**完整流程：**
1. agent 找到对应 task（按标题匹配 + 上下文）
2. 询问 outcome： "结果是什么？"  →  用户描述
3. agent 生成 CV 级别描述（动词开头 + 结果/影响）： "修复高优先级登录鉴权 bug，消除用户阻塞"
4. 询问 reflection： "有什么复盘想法？"（可选，不强迫）
5. 写入成就库（永久，append-only）
6. 提醒： "✨ 已沉淀进成就库，下个季度述职可以直接用"

**成功标准：** 90 秒内完成"完成+沉淀"，**比手动整理快 10 倍**。

### 场景 3：问局势 → 主动建议

**触发：** "我现在该干啥？" / "最近有什么事？" / "帮我排个序"

**完整流程：**
1. agent 调用 `snapshot` 看全部 task
2. 按 priority + due 排序
3. 推荐 Top 3 任务，每个带理由：
   - "最紧迫：修登录 bug 截止今天"
   - "被 block 可以放：演讲 PPT 在等设计稿"
   - "可推进：周报草稿（无截止、但周五要交）"
4. 标记"被 block 可以先放"的任务，让用户心理上卸担子

**成功标准：** 用户 5 秒内知道今天该做什么，**不用打开 app 翻找**。

### 场景 4：周报自动整理

**触发：** "帮我整理这周周报" / "周五了，写周报"

**完整流程：**
1. agent 查询本周成就库（since=周一）
2. 按项目分组
3. 用户选择模板： `产品周报` / `研发周报` / `OKR 周报` / `自定义`
4. 生成 Markdown 周报草稿
5. 用户对话微调： "把'修复 bug'那条放最上面" / "加一行'下周计划'"
6. 一键复制到剪贴板

**输出示例：**
```markdown
## 本周完成（2026-07-06 ~ 2026-07-12）

### Q3 产品迭代
- 修复高优先级登录鉴权 bug，消除用户阻塞
- 完成登录页改版，A/B 测试跳出率 -18%
- 主导 App 改版上线，DAU +5%

### 季度复盘
- 准备并主讲 Q2 复盘演讲，推动 2 项流程改进
```

**成功标准：** 比用户手动整理**快 5 倍**，从 30 分钟降到 5 分钟。

### 场景 5：述职材料生成

**触发：** "帮我写 Q3 述职" / "晋升答辩材料"

**完整流程：**
1. agent 查询 Q3 成就库（since=2026-07-01）
2. 只用 `cvStatus=ready` 的条目（保证真实）
3. 用户选择侧重： `领导力` / `专业能力` / `业务结果` / `综合`
4. 按 STAR 结构组织（背景/行动/结果）
5. 用户对话微调： "把'App 改版'那条加更多数据" / "加一段'团队协作'维度"
6. 生成 Markdown 述职材料
7. 导出 PDF（可选）

**输出示例：**
```markdown
## Q3 述职材料

### 核心成果

- **主导 App 改版项目**
  - 背景：原有 UI 跳出率偏高（行业平均 1.5x）
  - 行动：牵头视觉重设计 + 性能优化，跨 5 个团队协作
  - 结果：跳出率 -18%，DAU +5%，用户好评 +12%

- **修复高优先级登录 bug**
  - 背景：线上 P0 故障，影响 30% 用户
  - 行动：2 小时内定位 + 修复 + 灰度上线
  - 结果：消除用户阻塞，当日验证，零回滚
```

**成功标准：** 述职前 1 小时能完成，**过去要 1-2 天**。

## 4. 功能优先级（MoSCoW）

### Must Have（MVP v1.0 必须有）

- [ ] 对话式任务管理（场景 1-3）
- [ ] 本地优先存储（单机版零成本可用）
- [ ] Web 端可访问（浏览器打开即用）
- [ ] LLM 集成（Anthropic API + Ollama 本地降级 + 用户自定义 endpoint）
- [ ] 成就库 + CV/周报导出（场景 4-5）
- [ ] 项目/任务 CRUD + 优先级/截止日期/block
- [ ] 全局快捷键（macOS: Cmd+Shift+Space 唤起对话窗口）
- [ ] Onboarding（5 分钟首次引导）

### Should Have（v1.1 应该有）

- [ ] 飞书/微信机器人集成（个人用户的 IM）
- [ ] 跨设备同步（iCloud Drive / 自建后端）
- [ ] 周报/述职模板库（产品/研发/OKR/自定义）
- [ ] 浏览器扩展（捕获阅读片段，沉淀为成就原料）
- [ ] 语音输入（macOS 语音转文字）

### Could Have（v1.2 可以有）

- [ ] 移动端 PWA（iOS Safari 添加到主屏）
- [ ] 公开成就页（个人 brand 引流）
- [ ] 任务时间追踪（自动 + 手动）
- [ ] 子任务 / 任务模板
- [ ] 数据导入（从 Notion / Todoist / Things / 旧 skill 仓库导入历史任务）

### Won't Have（明确不做）

- ❌ 主动操作电脑（Computer Use 路线）
- ❌ 主动发邮件/IM
- ❌ 通用 AI 笔记（只做工作场景）
- ❌ 团队实时协作（交给飞书/钉钉）
- ❌ 知识图谱 / 双向链接
- ❌ 番茄钟 / 时间块管理
- ❌ 团队版（飞书/钉钉/Notion 已经做了）

## 5. 信息架构

### 5.1 数据模型（继承自 skill 形态，扩展为多端可访问）

```
Project (项目)
├── id: string (proj_xxx)
├── name: string
├── createdAt: date
└── archived: bool

Task (任务)
├── id: string (task_xxx)
├── project: project_id
├── title: string
├── status: "未开始" | "进行中" | "已完成"
├── priority: "高" | "中" | "低"
├── due: date (可选)
├── nextAction: string (下一步动作)
├── blocked: bool (被外部阻塞)
├── draft: bool (待用户确认)
├── createdAt: date
├── completedAt: date (完成后填)
└── checklist: [string] (子任务)

Achievement (成就) - append-only
├── id: string (done_xxx)
├── date: date (完成日期)
├── taskId: string (来源任务)
├── projectId: string
├── project: string (项目名快照)
├── title: string (任务标题快照)
├── outcome: string (用户描述的结果)
├── reflection: string (用户复盘，可选)
├── cv: string (agent 生成的 CV 描述)
├── cvStatus: "pending" | "ready"
└── tags: [string] (v1.1 标签系统)

CVExport (导出记录) - v1.1
├── id, type: "周报" | "述职" | "简历"
├── period: date range
├── content: markdown
└── createdAt
```

### 5.2 页面结构

```
主对话窗口（默认首页，80% 时间在这里）
  ├── 顶部：当前对话上下文（最近 10 条消息）
  ├── 中部：用户输入区（支持文字/语音）
  ├── 底部：快捷操作按钮
  │   ├── "新增项目" / "新增任务"
  │   ├── "我现在该干啥"
  │   ├── "整理周报"
  │   └── "述职材料"
  └── agent 回复带结构化卡片（任务列表/项目卡片/成就卡）

侧边栏（可折叠）
  ├── 今日聚焦（5 条最高优先级）
  ├── 项目列表
  │   └── 项目卡片（含任务数/未完成数）
  └── 任务列表
      ├── 全部 / 进行中 / 已完成
      └── 过滤：项目/优先级/blocked

成就库 Tab
  ├── 时间线视图（按日期分组）
  ├── 项目过滤
  ├── 状态过滤（pending/ready）
  └── 批量导出

导出 Tab
  ├── 周报生成器
  ├── 述职材料生成器
  └── 历史导出记录

设置 Tab
  ├── LLM 配置（API key / 模型选择 / Ollama 路径）
  ├── 同步（iCloud / 自建后端 / 关闭）
  ├── 通知（飞书/微信/系统通知）
  ├── 主题（亮/暗/跟随系统）
  └── 订阅状态
```

### 5.3 数据流

```
用户输入（对话/快捷键/语音/IM 机器人）
  → LLM 解析意图 + Function Calling
  → 调用拾光 API（CRUD）
  → 写入本地 SQLite
  → 实时推送 Web UI（WebSocket）
  → 触发通知（飞书/微信/系统通知 if 配置）
  → （如果启用云端同步）加密上传到云端
```

## 6. 关键交互

### 6.1 对话窗口（核心）

**触发：** 全局快捷键 `Cmd+Shift+Space`（macOS）/ `Win+Shift+Space`（Windows）

**界面：**
- 全屏弹窗，半透明背景
- 顶部：当前对话上下文（最近 10 条）
- 中部：用户输入区（支持文字/语音/截图粘贴）
- 底部：快捷操作按钮 + 输入框
- agent 回复带结构化卡片：
  - 任务列表卡片（带勾选/优先级 tag）
  - 项目卡片（带任务数）
  - 成就卡（带"沉淀成功"徽章）
  - 建议列表（"今天建议做..."）

**交互模式：**
- 打字输入（默认）
- 语音输入（点击麦克风，自动转文字）
- 截图粘贴（OCR + agent 解析）
- 快捷命令（`/log` `/week` `/cv` `/focus` 等）

### 6.2 任务列表（侧边栏）

**视图：**
- "今日聚焦"（Top 5 高优任务）
- "按项目"（项目分组）
- "全部"（平铺）
- "已完成"（成就库入口）

**快速操作：**
- 单击任务：查看详情 + 编辑
- 长按任务：弹操作菜单（完成/改期/block/拆子任务/删除）
- 拖拽任务：改优先级（v1.1）
- 双击：标记完成 + 触发"完成即沉淀"流程

### 6.3 成就库

**视图：**
- 时间线（按日期倒序）
- 项目过滤（多选）
- 状态过滤（pending / ready）
- 标签过滤（v1.1）
- 搜索（按标题/cv 内容）

**操作：**
- 单击：查看详情 + 编辑 cv
- 批量勾选 + 导出（生成周报/述职）
- 升 pending → ready（补充数据后）

### 6.4 导出流程

**周报：**
1. 选择时间范围（默认本周）
2. 选择模板（产品/研发/OKR/自定义）
3. agent 预览生成
4. 对话微调
5. 一键复制 / 下载 .md

**述职：**
1. 选择季度 / 自定义时间
2. 选择侧重维度
3. agent 预览
4. 对话微调（多次）
5. 下载 .md / .pdf

## 7. 技术架构

### 7.1 总体

| 层 | 技术选型 | 理由 |
|---|---|---|
| **后端** | Python 3.11+ / FastAPI / pydantic v2 | 复用 skill 形态的 cockpit 逻辑，Python 生态成熟 |
| **数据库** | SQLite（本地默认）+ PostgreSQL（云端） | 本地零配置，云端支持并发 |
| **前端** | Next.js 15 (App Router) / React 19 / TailwindCSS / shadcn/ui | React 生态成熟，SSR/CSR 灵活 |
| **LLM 集成** | Anthropic API（默认）+ Ollama（本地降级）+ OpenAI 兼容 | 用户可选，零锁定 |
| **存储同步** | iCloud Drive（轻量方案）+ 自建后端（订阅用户） | 两层覆盖 |
| **部署** | 单机版：pip install + 本地运行；云端版：FastAPI on fly.io | 单机版零运维 |
| **WebSocket** | FastAPI native + Next.js client | 实时推送 |

### 7.2 关键决策

**LLM 默认：** Anthropic Claude Sonnet 4.5
- 中文支持好
- Function calling 稳定
- Context window 200K（够用）
- 价格合理（输入 $3/M, 输出 $15/M）

**本地 LLM fallback：** Ollama + Qwen2.5-14B
- 中小模型中文好
- 16GB 内存可跑
- 量化后 8GB 可用

**数据加密：**
- 本地数据库可选加密（SQLCipher）
- 云端同步 E2E 加密（libsodium / age）
- 服务器不可读

**API Key 管理：**
- 用户自带 key（不中转）
- 只统计调用次数做限额
- 不存明文（macOS Keychain / 浏览器存储加密）

### 7.3 安全 / 隐私（这是底线）

- ✅ 默认本地存储，云端 opt-in
- ✅ 端到端加密：用户的成就数据即使上云也读不到
- ✅ LLM 调用脱敏：发送前去除 PII（手机号/邮箱/身份证号）
- ✅ 审计日志：所有 LLM 调用可追溯
- ✅ 数据导出/删除：一键导出所有数据 + 一键删除
- ✅ 离线可用：核心功能（CRUD + 成就库查看）不依赖网络

### 7.4 性能 / 可靠性目标

- 首屏加载 < 1s（本地）/ < 2s（云端）
- 对话响应 < 3s（含 LLM 调用）
- WebSocket 推送 < 100ms
- 客户端崩溃率 < 0.1%
- 数据零丢失（CRUD 都走事务）

### 7.5 与 skill 仓库的关系

**关键约束：原 skill 仓库 `CookieJobs/task-cockpit` 保持独立维护，不 fork、不 submodule。**

拾光不是 skill 的"产品化分支"，而是**形态不同的新产品**：
- skill 仓库：用 SKILL.md 触发的 LLM agent 通过 CLI 调用 cockpit.py
- 拾光：独立 web 产品，有自己的后端 + 前端 + 数据库

**代码层面：**
- 拾光**不复用** skill 的代码（不 fork、不 submodule）
- 拾光在重写时**严格继承** skill 的设计哲学：相同的数据模型、相同的状态机、相同的数据目录习惯
- 提供数据导入脚本：从 `~/.task-cockpit/` 导入历史数据到 `~/.shiguang/`

## 8. 商业模式

### 8.1 两层定价（v1.0，不做团队版）

| 版本 | 价格 | 包含 |
|---|---|---|
| **免费版** | ¥0 | 单机本地存储、AI 对话 100 次/月、基础功能 |
| **专业版（个人订阅）** | ¥19/月 或 ¥190/年 | 跨端同步、无限 AI 对话、所有模板、语音输入、浏览器扩展 |

### 8.2 转化路径

- **免费 → 专业：** 单设备限制 + AI 额度耗尽时弹窗
  - 触发时机：用户第 5 次达到额度时
  - 文案： "你已用完本月 AI 对话额度，开通专业版享无限对话"

### 8.3 不做的策略

- ❌ **不做"年付打折"**（用月付降低决策成本）
- ❌ **不做"功能阉割版免费"**（基础功能必须可用，不吊胃口）
- ❌ **不做"广告/数据销售"**（隐私是底线）
- ❌ **不做"免费试用 X 天"**（免费版直接给用，不诱导升级）
- ❌ **不做"团队版"**（飞书/钉钉/Notion 已经做了）

### 8.4 收入预测（保守）

- **6 个月：** 200 付费用户 × ¥19 = ¥3,800/月 = ¥45,600/年
- **12 个月：** 1,000 付费用户 × ¥19 = ¥19,000/月 = ¥228,000/年
- **18 个月：** 3,000 付费用户 × ¥19 = ¥57,000/月 = ¥684,000/年

### 9.1 自用期（0-3 个月）· dogfooding 优先

## 12. 成功指标

### 12.1 北极星指标

**每周活跃用户产出的"ready"成就数**

> 反映产品真实使用深度。log + write 都简单，但"准备好的成就"是用户的真实产出。

### 12.2 关键 KPI

| 阶段 | 指标 | 目标 |
|---|---|---|
| **激活** | 首次使用产出第一个成就 | < 5 分钟 |
| **留存** | D7 留存 | > 30% |
| **粘性** | 周活/月活 | > 50% |
| **转化** | 免费→付费转化率 | > 5% |
| **核心功能使用** | 周报/述职功能使用率 | > 50% MAU |
| **质量** | LLM 调用成功率 | > 95% |
| **稳定** | 客户端崩溃率 | < 0.1% |
| **NPS** | 净推荐值 | > 30 |

### 12.3 反指标（不要优化这些）

- ❌ DAU（堆功能提频会伤害产品定位）
- ❌ 成就数（堆数会伤害质量）
